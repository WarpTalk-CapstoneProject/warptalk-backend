# WT-121 Device Flow and Control Semantics

## 1. High-level Architecture

```mermaid
flowchart LR
  MIC["Real Microphone"] --> CAPA["Capture: Local Input"]
  CAPA --> STT1["STT Pipeline"]
  STT1 --> TR1["Translation"]
  TR1 --> TTS1["TTS Output"]
  TTS1 --> VMIC["Virtual Microphone Output"]
  VMIC --> APP1["Third-Party Meeting App"]

  APP1 --> VCABLE["Virtual Speaker/Cable Output"]
  VCABLE --> CAPB["Capture: Remote Input"]
  CAPB --> STT2["STT Pipeline"]
  STT2 --> TR2["Translation"]
  TR2 --> TTS2["TTS Output"]
  TTS2 --> SPK["Real Speaker/Headphones"]
```

## 2. Role Separation Rules

- Real microphone MUST only serve local capture input.
- Virtual microphone MUST only serve outbound translated speech to meeting app.
- Virtual speaker/cable MUST only serve inbound meeting audio capture source.
- Real speaker/headphones MUST only serve local listening output.

## 3. Loop-Prevention Policy

- Every audio frame carries direction tag (`local_to_remote` or `remote_to_local`).
- Capture stages reject frames marked as generated by local TTS output route.
- Any output device that equals current capture source is blocked and triggers configuration warning.

## 4. Mute Semantics

- `local_to_remote_mute = true`: suppress TTS output to virtual microphone only.
- `remote_to_local_mute = true`: suppress TTS output to real speaker/headphones only.
- Muting does not require stopping transcript ingestion unless operator explicitly pauses capture.

## 5. Latency Bands and Actions

- `normal`: end-to-end latency under target threshold, no action.
- `warning`: latency above soft threshold, show warning and suggest shorter scripted turns.
- `critical`: latency above hard threshold, suggest transcript-first fallback and pause long-form speech.

## 6. Error States and Recovery

- `permission_denied`: request OS permission re-grant, reinitialize capture.
- `device_missing`: prompt rebind for the missing role, rerun sanity check.
- `pipeline_degraded`: continue transcript path where possible, limit TTS path until recovered.
